technical-emergent-of — openFrameworks app
==========================================

GitHub is the chronicle: clone walhimer-studio/openframework-puredata, then work inside
technical-emergent-of/. You do not need to copy this folder into OF/apps/myApps/.

1. Install openFrameworks 0.12.x for macOS (https://openframeworks.cc/download/).

2. Set your OF install path in Project.xcconfig: OF_PATH and the #include line for
   CoreOF.xcconfig (same folder). Set OF_ROOT in config.make to match if you use make.

3. ofxOsc is vendored in addons/ofxOsc (committed) so Xcode finds sources after clone.
   For command-line make, symlink or copy that folder to YOUR_OF_ROOT/addons/ofxOsc
   (or keep a full OF tree and point OF_ROOT at it).

4. Build (from technical-emergent-of/):
     xcodebuild -project technical-emergent-of.xcodeproj -scheme "technical-emergent-of Debug" -configuration Debug build
   Or open technical-emergent-of.xcodeproj in Xcode.

5. Build and run. Place piano WAV samples under:
     bin/data/sounds/
   Required filenames (Salamander Grand or your own), matching the HTML prototype:
   A2.wav Ds2.wav Fs2.wav C2.wav C3.wav Ds3.wav Fs3.wav A3.wav C4.wav Ds4.wav Fs4.wav A4.wav
   C5.wav Ds5.wav Fs5.wav A5.wav C6.wav Ds6.wav

6. Keys:
   o      Toggle OSC to Pd vs local WAV playback
   Space  New random seed (non-deterministic)
   R      Regenerate same seed
   N      Increment seed by 1 and regenerate

Seeded RNG matches github.com/walhimer-studio/EmergentDNA SeedRng (sin-based next()).

Pure Data (pd/)
---------------
See pd/README.txt. Run pd/setup-samples.sh, install iemnet (Deken), open pd/main-osc.pd
before or after starting the oF app. OSC: oF -> Pd port 9000 /from-of, Pd -> oF port 9001 /to-of-rms.
